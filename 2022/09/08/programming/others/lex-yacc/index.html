lex 和 yacc 是一对帮助编写其他程序的程序。lex 和 yacc 的输入描述了希望最终程序如何工作，输出是 C 编程语言的源代码；您可以编译此源代码以获得按照您最初描述的方式工作的程序。

LEX 是 lexical analyzer 的缩写，lex 生成的 C 代码分析输入并将其分解为 token。YACC 全称 Yet Another Compiler-Compiler，yacc 生成的 C 代码解释 lex 代码获得的 token。

lex 和 yacc 工作流程

lex 的输出

lex 的目标是为名为 yylex() 的 C 函数生成代码。该函数在没有参数的情况下被调用，并返回一个int 值。到达文件结尾时返回值 0；否则，yylex() 返回一个值，指示找到了哪种 token。

lex 还创建了两个重要的外部数据对象：

  • 名为 yytext 的字符串，从流 yyin 中读取，默认为标准输入 stdin
  • 名为 yyleng 的整数变量，给出了 yytext 字符串中数目

在大多数 lex 程序中，yytext 中的 token 具有必须计算并传递给支持程序的关联值。按照惯例，yacc 将此数据对象命名为yylval。例如，如果 yylex() 读取一个整数 token，则说明 yytext 包含组成整数的数字字符串，而 yylval 通常存储整数的实际值。默认情况下，yylval 被声明为 int，但有一些方法可以更改此默认值。

yacc 的输出

yacc 的目标是为名为 yyparse() 的 C 函数生成代码。yyparse() 函数调用 yylex() 从标准输入中读取一个 token，直到文件结束。yyparse() 使用 yylex() 的返回值来计算获得的每个 token 的类型，并在每种情况下使用yylval 作为该 token 的实际值。

yyparse() 函数在没有任何参数的情况下被调用，如果解析的输入是有效的，则函数的结果为 0，如果输入包含任何类型的错误，则结果为 1。

字符和正则表达式

字符串用 "" 包括，且支持 \n\t 等转义字符。相关语法与之前写的 正则表达式 | Python/Julia/Shell 规则类似，其中 \d 之类的匹配使用 POSIX 字符（参看 shell 的部分）。举个例子，^"xxx"$ 匹配以 xxx 开头和结尾的字符串。

留意的是，正则匹配时，字符串不会看成一个整体，比如 "ab"+ 会匹配 ab 和 abb 等，而不会将 ab 当成整体取匹配 abab。当需要处理成整体时，可使用分组，比如 ("ab")+。

注：和通常的正则匹配有区别的是，^() 等算符要写在字符串外边。
